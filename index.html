  7. Where a database is to be recovered from hd crash we use check pt method of log – recovery . a check –pt is a special log record , the main sequence in this recovery : Output onto stable storage all log records residing in main memory
    output all modified buffer blocks onto disk
    output onto stable storage a log record check pt
    No database modifications can occur where a check pt in progress
    we suppose that a hd crash occurs where a transaction executing . the log is consulted backwds till sees the transaction – commit ,
    log is still consulted backwds till sees the check pt , log is still consulted backwds till sees the transaction start --- that identifies the transaction . undo or redo operations can be performed on this transaction and any others For concurrent transaction processing fuzzy check pt is used
  8. A transaction is a unit of program execution that accesses and or possibly updates various data items . the Acid properties of a transaction are :
    Atomicity --- either transaction executes in entirety or none executes at all
    Consistency --- state of database remains same both before & after transaction despite system changes
    Isolation--- despite concurrent execution of transactions each transaction proceeds independently of others despite system changes
    Durability ---- any modifications made to database as a result of transaction persist even if there are system changes
    e.gs we have a schedule involving 2 accts a and b, the initial values in a and b are $1000 and $ 2000 .

    a transaction occurs that transfers $ 50 from a -b as a result final values in a and b becomes $ 950 and $ 2050 .
    clearly a consistent state as sum a +b is preserved both before & after transaction. Now we suppose that a hd crash occurs after write – a
    committed but before write – b committed , so, now the final values in a and b becomes $ 950 and $ 2000 clearly inconsistent state
    as sum a +b is no longer preserved both before & after transaction . even if system crash did not occurs and transaction proceeded to completion ,
    this inconsistent state would not have been visible except during occurrence of transaction.
    This highlights Atomicity . isolation is preserved . durability is preserved through recovery mgmt.. component of transactions
